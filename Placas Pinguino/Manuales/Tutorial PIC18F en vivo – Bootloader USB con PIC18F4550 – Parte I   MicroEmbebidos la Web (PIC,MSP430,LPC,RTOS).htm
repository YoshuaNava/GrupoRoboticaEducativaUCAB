Tutorial PIC18F en vivo – Bootloader USB con PIC18F4550 – Parte I

mayo 29, 2013   PIC18F EN VIVO, TUTORIALES   7 Comments

Hemos vuelto a la carga con el C18! Pensar que han pasado 5 años desde el último post y aún Microchip sigue sin superar su propio C18 (hace 1 año han lanzado su XC18 pero es una mie!/[]#$a así como lo es su mplabx). En esta oportunidad vamos realizar un super tutorial sobre la implementación de un bootloader que ofrece de forma gratuita Microchip, con la ligera adaptación realizada por mí para que trabaje con una placa que tiene el PIC18F4550. Con este bootloader podrás grabar códigos sin necesidad de un hardware adicional como un grabador y todo por USB. Esta nueva serie de tutos mezcla el C18, el PIC18F4550, y el software PDFSUSB de microchip. Este tutorial constará de 3 partes y en esta primera parte ofreceremos todos los archivos necesarios y una breve explicación de cómo funciona el bootloader para la placa pinguino 18F4550.

Materiales para el Tutorial Boot C18

Vamos a necesitar los siguientes elementos de harware y software:

  • Placa 18F4550, ver enlace de cómo construirlo (de preferencia nomás).
  • Cualquier grabador para PICs que soporte el 18F4550 (yo uso el pickit2).
  • MPLAB IDE (de preferencia las versiones 8.xx, yo uso la 8.60).
  • Compilador C18 (yo uso la v3.40, con otras superiores debería ser lo mismo).
  • El firmware del bootloader BootV5 (4shared, boxnet). He realizado una actualización, la cual permite utilizar el bootloader sin conexión USB se llama BootV6 (4shared, boxnet), más detalles al final de este post.
  • El firmware de aplicación compatible con el bootloader (4shared, boxnet). He tachado estos links porque contienen el archivo Demo1ForBoot.rar que pide una carpeta externa innecesaria (error mío). Utilicen estos links donde está la version  Demo1ForBootV2.rar (4shared, boxnet) corregida y lista para compilar sin complicaciones.
  • El software para utilizar el bootloader desde la PC (4shared, boxnet).
  • El driver “inf” para que la PC identifique al hardware (4shared, boxnet).

Implementación del Bootloader

Para probar el bootloader, seguir estos pasos:

PASO1. Tener todos los materiales y archivos mencionados arriba.

PASO2. Abrir el proyecto BootV5 y compilarlo (archivo BootV5.rar).

PASO3. Usando el grabador que tengan, importar el archivo BootV5.hex y grabarlo en el PIC18F4550 (pueden sacar el micro de la placa pinguino o sino también puden usar el bus ICSP de la placa pinguino y conectarlo al bus ICSP que todo programador de PICs tiene disponible).

PASO4. Ejecutar el programa PDFSUSB.exe (archivo originalpdfsusb.rar).

PASO5. Conectar la placa pinguino a la PC mediante un cable USB.

PASO6. El led rojo RUN del pinguino comenzará a parpadear a 500mseg durante 10 segundos, luego de ese tiempo se ejecutará el código de la aplicación (como viene de ser programado recién toda la memoria está limpia por lo tanto el micro se reseteará).

PASO7. En la pestaña Bootload Mode del PDFSUSB.exe, hay un casillero vacio y le dan click en su botón para ver su contenido y encontrarán: PICDEM FS USB 0 (Boot), y lo seleccionan. Entonces verán como se activan los otros botones.

PASO8. En simultáneo, el led rojo RUN comenzará a parpadear a 150mseg indicando que se encuentra en modo bootloader y a la espera de comandos.

PASO9. Abrir el proyecto Demo1ForBoot y compilarlo (archivo Demo1ForBoot.rar). Este firmware de aplicación tan sólo hará parpadear un led por el pin RB7 del PIC18F4550.

PASO10. Usando el PDFSUSB, importar el archivo Demo1ForBoot.hex con el botónLoad HEX File, luego darle al botónErase Device seguido del botón Program Device para ser grabado en el PIC18F4550. Finalmente usar el botón Execute para ver el programa de aplicación Demo1ForBoot corriendo.

La siguiente vez que reinicien la placa, después de los 10 segundos se irá a ejecutar la aplicación existente. En adelante para grabar otros firmwares de aplicación hacer desde el paso 4.
